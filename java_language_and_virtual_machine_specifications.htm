Java Type System

Primitive Types

Reference Types

Built-in, non-object types that store actual values directly in memory (stack if local/heap if in object)

Types that store references (memory addresses) to objects on the heap

8 in total - boolean, byte, short, char, int, long, float, double

Unlimited number; four main categories - classes, interfaces, arrays, and type variables

Always immutable - values held in a variable are swapped, the value itself remains fixed

Mutable - ArrayList, StringBuilder, most custom classes

Immutable - String, Integer, LocalDateTime (once created, state cannot be changed/mutated)
